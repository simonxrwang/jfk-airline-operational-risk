\begin{frame}{本章在小组报告中的位置}
  % Speaking cue: 30--35 seconds.
  \begin{itemize}
    \item 前一章已经完成行业背景、核心风险因素、SWOT 与 PEST 分析。

\begin{frame}{变量设计：服务 frequency--severity 模型}
  % Speaking cue: 40 seconds.
  \begin{columns}[T,totalwidth=\linewidth]
    \begin{column}{0.33\linewidth}
      \textbf{频率变量}
      \begin{itemize}
        \item 总到达航班量
        \item 15 分钟以上延误
        \item 取消与改降
      \end{itemize}
    \end{column}
    \begin{column}{0.33\linewidth}
      \textbf{严重度变量}
      \begin{itemize}
        \item 总延误分钟
        \item 平均每次延误分钟
        \item 分原因延误分钟
      \end{itemize}
    \end{column}
    \begin{column}{0.33\linewidth}
      \textbf{风险分类变量}
      \begin{itemize}
        \item Airline 与 Late Aircraft
        \item NAS
        \item Weather 与 Security
      \end{itemize}
    \end{column}
  \end{columns}
  \vspace{0.8em}
  \centering
  \large
  数据结构先拆清楚，后续模型才可以分别回答“发生多频繁”和“发生后多严重”。
\end{frame}

\begin{frame}{结论：为什么这部分对后续模型重要}
  % Speaking cue: 35--40 seconds.
  \begin{enumerate}
    \item 清洗后的核心数据让频率模型和严重度模型有明确输入。
    \item 原因分类把 BTS 字段转化为课程中的 operational risk blocks。
    \item 月度时间序列为 normal/disrupted 状态划分和 Markov 转移估计打基础。
    \item 图表和 dashboard 让模型结果可以被业务语言解释，而不是停留在公式层面。
  \end{enumerate}
